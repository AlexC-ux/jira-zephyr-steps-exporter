# JIRA Zephyr Test Steps Export

> Экспорт шагов тестов из Jira Zephyr в формат ODS (OpenDocument Spreadsheet)

## 📋 Описание

Скрипт для автоматической выгрузки шагов тестовых кейсов, привязанных к задачам Jira по заданному JQL-запросу. Результат сохраняется в файл формата ODS (совместим с Excel, LibreOffice Calc и другими табличными редакторами).

Использует:
- Jira REST API для получения задач
- Zephyr for Jira Server API для получения тестовых результатов и шагов
- JSDOM для парсинга HTML-содержимого
- SheetJS (xlsx) для генерации таблиц

## 🚀 Возможности

- Экспорт шагов тестов по JQL-фильтру
- Автоматическая пагинация (обработка большого количества задач)
- Парсинг HTML в описаниях шагов (удаление тегов, нормализация пробелов)
- Объединение ячеек в ODS для удобства чтения (Task Key, Test Result Key)
- Сортировка по Task Key → Test Result Key → Step Index
- Подробное логирование процесса
- Генерация отчета в формате ODS

## 📋 Требования

- Node.js версии 22 или выше
- npm (обычно устанавливается вместе с Node.js)

## ⚙️ Установка и настройка

### 1. Установка зависимостей

```bash
npm install
```

### 2. Настройка окружения

Создайте файл `.env` на основе примера:

```bash
copy .env.example .env
```

Или вручную создайте файл `.env` со следующими параметрами:

```env
jira_url="https://jira.yourcompany.ru"     # URL сервера Jira (без слеша в конце)
jira_username="your-email@company.com"      # Email или логин для авторизации
jira_password="your-api-token-or-password"  # Пароль или API token
jira_task_load_chunk=100                    # Размер пачки задач (1-1000)
jira_jql="project = PROJ AND status = Done" # JQL запрос для поиска задач
outpit_file_name="test_steps_export"        # Имя выходного файла (без расширения)
```

#### Описание переменных

| Переменная | Обязательная | Описание |
|------------|-------------|----------|
| `jira_url` | ✅ | URL вашего сервера Jira (например, `https://jira.company.ru`) |
| `jira_username` | ✅ | Логин (email) для авторизации в Jira |
| `jira_password` | ✅ | Пароль или API token для доступа к Jira API |
| `jira_task_load_chunk` | ❌ | Количество задач за один запрос (по умолчанию: 100, максимум: 1000) |
| `jira_jql` | ✅ | JQL-запрос для поиска задач (например, `project = PROJ AND status = Done`) |
| `outpit_file_name` | ❌ | Имя файла без расширения (по умолчанию: timestamp) |

#### 💡 Советы по JQL

Примеры полезных JQL-запросов:
- `project = PROJ` - все задачи проекта
- `project = PROJ AND status = Done` - завершенные задачи
- `project = PROJ AND assignee = currentUser()` - задачи текущего пользователя
- `project = PROJ AND updated >= -30d` - обновленные за последние 30 дней
- `key = PROJ-123` - конкретная задача

## 📊 Выходной формат

### Структура файла ODS

Файл содержит один лист `Test Steps` со следующими колонками:

| Колонка | Описание |
|---------|----------|
| **Task Key** | Ключ задачи Jira (например, `PROJ-123`) |
| **Test Result Key** | Ключ тестового результата (например, `TESTR-456`) |
| **Step Index** | Номер шага (1, 2, 3...) |
| **Action (Description)** | Действие (описание шага теста) |
| **Expected Result** | Ожидаемый результат шага |

### Особенности форматирования

- **Объединение ячеек**: Для удобства чтения значения Task Key и Test Result Key объединяются по группам
- **HTML парсинг**: Все HTML-теги автоматически удаляются, пробелы нормализуются
- **Сортировка**: Данные отсортированы по Task Key → Test Result Key → Step Index

### Пример данных

| Task Key | Test Result Key | Step Index | Action | Expected Result |
|----------|-----------------|------------|--------|-----------------|
| PROJ-100 | TR-100 | 1 | Открыть страницу входа | Отображается форма входа |
| PROJ-100 | TR-100 | 2 | Ввести логин и пароль | Поля заполнены |
| PROJ-100 | TR-100 | 3 | Нажать "Войти" | Пользователь авторизован |
| PROJ-100 | TR-101 | 1 | Открыть профиль | Отображается страница профиля |
| PROJ-101 | TR-102 | 1 | Перейти в настройки | Отображается страница настроек |

## 🔧 Технические детали

### Используемые API

#### Jira REST API
```
GET /rest/api/2/search?jql={jql}&startAt={start}&maxResults={size}&expand=changelog
```

#### Zephyr for Jira Server API
```
GET /rest/tests/1.0/issue/{issueId}/tracelinks?maxResults=50&startAt=0
GET /rest/tests/1.0/testresult/{testResultKey}?fields={fields}
```

### Основные функции

| Функция | Описание |
|---------|----------|
| `getIssuesByJql(jql, startAt)` | Получение задач по JQL с пагинацией |
| `getTestResultsByIssue(issueId)` | Получение тестовых результатов для задачи через traceLinks |
| `getTestScriptResults(testResultKey)` | Получение шагов тестового результата |
| `formatFieldValue(value)` | Очистка HTML и нормализация текста |
| `calculateMerges(data)` | Вычисление диапазонов для объединения ячеек |

### Обработка ошибок

- Ошибки HTTP запросов к Jira вызывают выброс исключения с описанием
- Ошибки при получении тестовых данных логируются, но не останавливают выполнение
- Критические ошибки приводят к завершению скрипта с кодом 1

### Поток выполнения

```
1. Загрузка конфигурации из .env
2. Создание выходной папки output/
3. Пагинационная загрузка задач по JQL
4. Для каждой задачи:
   - Получение testResult через traceLinks
   - Для каждого testResult: получение шагов
   - Форматирование и добавление в массив
5. Сортировка всех данных
6. Генерация ODS файла с объединенными ячейками
7. Вывод статистики
```

## 🐛 Известные ограничения

- Максимальный размер пачки задач: 1000 (ограничение Jira API)
- Скрипт обрабатывает только Zephyr for Jira Server API (не Cloud)
- Требуется наличие прав на чтение всех задач по JQL запросу


### Решение проблем

**Ошибка авторизации (401):**
- Проверьте правильность логина и пароля
- Убедитесь, что у вас есть права на чтение задач по JQL
- Попробуйте использовать API token вместо пароля

**Нет данных в результате:**
- Проверьте JQL запрос в интерфейсе Jira
- Убедитесь, что задачи имеют привязанные тестовые кейсы
- Проверьте логи на наличие предупреждений

**Ограничение по количеству запросов:**
- Увеличьте значение `jira_task_load_chunk`
- Добавьте задержки между запросами в коде (строка 356)
